# AWS Snow Family 深度拆解（SAP备考进阶版）
Snow Family 是 AWS 针对「带宽不足场景」的「离线海量数据迁移王牌」，核心解决“TB/PB/EB级数据在线迁移要数月”的痛点——SAP考试重点考「三种设备的容量区分、场景选型、核心特性对比」，甚至会结合 DataSync 考“在线vs离线”的选择。下面用“搬家卡车”的通俗逻辑，逐个拆解设备、流程、考点，帮你彻底吃透！


## 一、先重申核心定位（备考必背）
Snow Family 的本质是「AWS 提供的物理数据迁移设备」—— 相当于“AWS 派卡车到你家门口装数据，拉回 AWS 数据中心后上传到云存储”，全程离线，不依赖公网带宽。  
核心口诀：“带宽不够，Snow 来凑；小量 Cone，中量 Ball，海量 Mobile”。


## 二、三种设备深度拆解（SAP必考！）
按“容量从小到大”排序，每个设备用“通俗定位+核心特性+考点细节+场景”讲透，重点记「容量对应场景」和「独有的特性」：

### 1. Snowcone（小雪球）—— “便携式小面包卡车”
#### 通俗定位
针对「小规模离线数据迁移+边缘计算场景」的“手掌大小设备”，适合偏远地区、分支机构的小体量数据（比如野外作业数据、偏远门店备份）。

#### 核心特性（SAP考点）
- 容量：两种规格——**8TB（存储优化型）** 和 **14TB（存储优化型）**，注意：这里的容量是“可用存储”，不是物理容量；
- 物理形态：极其便携（手掌大小，重约400克）、低功耗（可插普通电源或电池供电），适合无稳定电源的边缘场景；
- 核心功能：
  1. 数据迁移：支持 NFS/SMB 协议（和 DataSync 源协议一致），本地拷贝数据到设备，寄回 AWS 后自动上传到 S3/EFS；
  2. 边缘计算（可选）：部分型号内置处理器，支持运行 Lambda 函数、容器化应用（比如在偏远地区做数据预处理，再迁移到云端）；
- 加密：全程 AES-256 加密（设备本身加密+数据传输加密），符合合规要求（考试必记！）。

#### 适用场景
- 偏远地区小体量数据迁移（比如山区电站10TB监控数据）；
- 边缘场景数据采集+预处理（比如油田、矿山的传感器数据）；
- 临时备份迁移（比如分支机构突发断电，需要紧急备份数据到AWS）。

#### 考试陷阱
- 错误选项：“Snowcone 适合 PB 级数据迁移”“Snowcone 不支持加密”“Snowcone 没有边缘计算功能”；
- 正确选项：“Snowcone 容量为 8TB/14TB，适合小规模离线迁移”“Snowcone 支持 NFS/SMB 协议”“Snowcone 支持边缘计算和 AES-256 加密”。

### 2. Snowball Edge（雪球边缘）—— “企业级厢式货车”
#### 通俗定位
针对「企业级中大规模数据迁移（TB/PB级）」的主力设备，是 Snow Family 中最常用的型号，兼顾“数据迁移”和“边缘计算”双重能力。

#### 核心特性（SAP高频考点）
- 容量：两种规格——**80TB（存储优化型）** 和 **100TB（存储优化型）**，可用存储约为标称值（无大量冗余）；
- 物理形态：中型设备（约一台微波炉大小），需外接电源，支持机架式部署（适合数据中心环境）；
- 核心功能（考试重点！）：
  1. 数据迁移：支持 NFS/SMB/iSCSI 协议（覆盖文件/块存储），可并行拷贝数据（迁移速度比 Snowcone 快10倍以上）；
  2. 边缘计算（核心差异点）：内置高性能处理器（比如 Intel Xeon），支持运行 EC2 实例、容器、数据库（如 MySQL），可在边缘做数据处理（比如工厂生产数据实时分析，再批量迁移到云端）；
  3. 高可用性：支持设备级冗余（比如双硬盘备份），避免数据丢失；
- 加密：同样 AES-256 加密，设备自带加密密钥，仅用户和 AWS 能解密。

#### 适用场景
- 企业数据中心 TB/PB 级数据迁移（比如本地文件服务器50TB数据、数据库备份100TB数据）；
- 边缘计算+迁移联动（比如零售连锁门店的销售数据本地分析后，通过 Snowball Edge 批量迁移到 AWS）；
- 替代小型边缘数据中心（比如偏远地区的医院，用 Snowball Edge 运行医疗系统，同时备份数据到云端）。

#### 考试陷阱
- 错误选项：“Snowball Edge 仅支持数据迁移，不支持边缘计算”“Snowball Edge 容量为 EB 级”“Snowball Edge 不支持 iSCSI 协议”；
- 正确选项：“Snowball Edge 是企业级离线迁移主力，支持 TB/PB 级数据”“Snowball Edge 支持 EC2 实例和容器运行（边缘计算）”“Snowball Edge 支持 NFS/SMB/iSCSI 协议”。

### 3. Snowmobile（大雪橇）—— “超大型半挂卡车”
#### 通俗定位
针对「超大规模数据迁移（EB级，1EB=1024PB）」的“移动数据中心”，是 AWS 最大的离线迁移设备，专门解决“数据中心整体迁移”的痛点。

#### 核心特性（SAP冷门但必考！）
- 容量：单辆车容量高达 **100PB**（可用存储），相当于1000个 1TB 硬盘的总量；
- 物理形态：半挂卡车大小（长约18米），自带独立电源、空调、安保系统（防 theft、防物理损坏）；
- 核心功能（考试重点在“专属服务”）：
  1. 数据迁移：支持高速光纤接口（100Gbps），可对接本地数据中心的 SAN/NAS 存储，批量拷贝 EB 级数据（比如大型银行、科研机构的全量数据中心迁移）；
  2. 专属服务：AWS 提供「专人押运」（24小时安保）、「上门部署」（AWS 工程师到现场协助对接本地存储）、「全程监控」（实时跟踪设备位置和数据状态）；
- 加密：端到端 AES-256 加密，密钥由用户管控，AWS 无权访问原始数据。

#### 适用场景
- 超大型企业数据中心整体迁移（比如某电信运营商 500PB 用户数据迁移到 AWS）；
- 科研机构海量数据迁移（比如天文台的天文观测数据、基因测序机构的 DNA 数据，容量达 EB 级）；
- 合规要求的大规模数据迁移（比如金融机构的历史交易记录，需物理隔离传输，避免公网风险）。

#### 考试陷阱
- 错误选项：“Snowmobile 是在线迁移工具”“Snowmobile 容量为 PB 级”“Snowmobile 无需 AWS 专人押运”；
- 正确选项：“Snowmobile 用于 EB 级超大规模数据迁移”“Snowmobile 是半挂卡车大小的移动数据中心”“Snowmobile 由 AWS 专人押运，提供上门部署服务”。


## 三、三种设备核心对比表（SAP真题速查）
| 对比维度         | Snowcone                          | Snowball Edge                          | Snowmobile                          |
|------------------|-----------------------------------|---------------------------------------|-------------------------------------|
| 容量范围         | 8TB/14TB（TB级，小规模）          | 80TB/100TB（TB/PB级，中大规模）       | 100PB/车（EB级，超大规模）           |
| 物理形态         | 手掌大小，便携低功耗              | 微波炉大小，机架式部署                | 半挂卡车，移动数据中心              |
| 核心功能         | 数据迁移+基础边缘计算（Lambda）   | 数据迁移+高级边缘计算（EC2/容器/数据库） | 超大规模数据迁移（无边缘计算）       |
| 支持协议         | NFS/SMB                           | NFS/SMB/iSCSI                         | 高速光纤接口（对接SAN/NAS）          |
| 适用场景         | 偏远地区小规模迁移、边缘采集      | 企业TB/PB级迁移、边缘计算联动         | 数据中心整体迁移、EB级海量数据       |
| 考试关键词       | 便携、小规模、边缘采集            | 企业级、TB/PB、边缘计算              | EB级、超大规模、专人押运            |


## 四、关键补充：Snow Family 迁移流程（SAP架构题考点）
不管哪种设备，迁移流程都类似，核心是“申请→寄设备→拷贝→寄回→上传”，记住这个流程能应对架构题：
1. **申请设备**：在 AWS 控制台提交 Snow 设备申请，指定设备类型、容量、目标存储（S3/EFS）；
2. **AWS 寄设备**：AWS 把加密后的设备寄到用户指定地址（通常1-3个工作日）；
3. **本地拷贝数据**：用户收到设备后，通过 NFS/SMB/iSCSI 协议把本地数据拷贝到设备（可通过 AWS 提供的客户端工具操作）；
4. **寄回 AWS**：拷贝完成后，用户联系物流寄回 AWS 指定的数据中心；
5. **AWS 上传数据**：AWS 收到设备后，解密并把数据上传到用户指定的 AWS 存储（S3/EFS），上传完成后通知用户；
6. **设备销毁/回收**：用户确认数据无误后，AWS 会物理销毁设备的存储介质（或格式化后回收），确保数据安全。

### 考点提醒：
- 迁移流程的“离线本质”：数据从未经过公网，全程物理传输，适合安全敏感数据；
- 数据安全：设备加密、密钥管控、物理销毁，这三个点是 SAP 考试中“合规场景”的高频考点。


## 五、与在线工具（DataSync）的对比（SAP对比题必考）
| 对比维度         | Snow Family（离线）                | DataSync（在线）                      | 考试选型口诀                          |
|------------------|-----------------------------------|---------------------------------------|---------------------------------------|
| 传输方式         | 物理设备离线传输                  | 公网/DX/VPN 在线传输                  | 带宽不足选 Snow，带宽充足选 DataSync  |
| 数据规模         | TB/PB/EB 级（海量数据）            | GB/TB 级（中小规模数据）              | 海量数据选 Snow，中少量选 DataSync    |
| 迁移速度         | 取决于设备容量和拷贝速度（比如 Snowball Edge 拷贝100TB 约1-2天） | 取决于带宽（1Gbps 带宽迁移1TB 约2.5小时） | 带宽低时 Snow 更快，带宽高时 DataSync 更快 |
| 适用场景         | 偏远地区、带宽极低、安全敏感数据   | 城市地区、带宽充足、周期性迁移        | 偏远/低带宽/敏感数据→Snow；城市/高带宽→DataSync |
| 成本             | 设备租赁+物流费（TB级成本高于 DataSync） | 数据传输费（按GB计费，成本较低）      | 小量高频选 DataSync，大量低频选 Snow  |

### 真题示例：
某企业位于偏远山区，本地带宽仅 1Mbps，需要迁移 50TB 数据到 AWS S3，应选择哪种工具？
- 选项 A：AWS DataSync；
- 选项 B：Snowcone；
- 选项 C：Snowball Edge；
- 选项 D：Snowmobile。
- 答案：C（解析：带宽极低→离线迁移，50TB 属于 TB 级中大规模→Snowball Edge；Snowcone 容量不够，Snowmobile 容量过剩）。


## 六、考试陷阱大全（必记！）
1. 容量陷阱：“Snowcone 适合 PB 级迁移”“Snowball Edge 是 EB 级”“Snowmobile 是 PB 级”—— 记住“Cone（TB）→Ball（TB/PB）→Mobile（EB）”；
2. 功能陷阱：“Snowball Edge 不支持边缘计算”“Snowcone 支持 EC2 实例”“Snowmobile 有边缘计算功能”—— 只有 Snowball Edge 支持高级边缘计算，Snowcone 仅基础边缘计算，Snowmobile 无；
3. 在线离线陷阱：“Snowmobile 是在线迁移工具”“DataSync 是离线迁移工具”—— Snow 全系列是离线，DataSync 是在线；
4. 协议陷阱：“Snowball Edge 不支持 iSCSI 协议”“Snowcone 支持 iSCSI 协议”—— Snowball Edge 支持 NFS/SMB/iSCSI，Snowcone 仅支持 NFS/SMB；
5. 加密陷阱：“Snow Family 不支持加密”—— 所有 Snow 设备都支持 AES-256 加密。


## 七、备考总结（新手必背）
1. 选型口诀：“小量 Snowcone（8/14TB），中量 Snowball（80/100TB，带计算），大量 Snowmobile（100PB/车，EB级）”；
2. 核心考点：容量对应场景、Snowball Edge 的边缘计算功能、Snowmobile 的专人押运和 EB 级、所有设备支持 AES-256 加密、离线迁移本质；
3. 对比题技巧：看到“带宽不足”“偏远地区”“TB/PB/EB 级”→ 选 Snow Family；看到“带宽充足”“中少量数据”“周期性迁移”→ 选 DataSync；
4. 陷阱题重点：排除容量对应错误、功能错误、在线离线错误的选项。

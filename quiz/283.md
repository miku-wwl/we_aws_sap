#### 📝 [283/529] S3 Object Lambda 动态数据转换

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 用户请求 S3 对象时需要动态转换。
* **需求：** 根据用户类型返回不同格式数据。
* **约束：** 不能存储多个副本，实时转换。
* **性能要求：** 低延迟。

**2. ⚡ 秒杀思路**
* **数据转换时机：**
    * **上传时转换:** 存储多版本，空间浪费。
    * **请求时转换:** 动态处理，按需生成。
* **S3 Object Lambda 特性：**
    * **Lambda 集成:** 通过 Lambda 函数处理对象。
    * **透明代理:** 用户访问 S3 端点，自动触发转换。
    * **无存储开销:** 不需要存储转换后的副本。
* **实施方案：**
    * **选项 A (预处理):** 存储多个版本。
    * **选项 B (CloudFront + Lambda@Edge):** 复杂度高，需要管理缓存。
    * **选项 C (S3 Object Lambda):** 
        * 透明的实时转换。
        * 集成 S3 访问点。
        * 简化架构。
    * **选项 D (API Gateway):** 需要代理整个 S3 访问。
* **选择：** S3 Object Lambda 是专门为此设计的功能。

**3. ✅ 正确选项解析 (选项 C)**
* **S3 Object Lambda:** 请求级别的数据转换，无需存储副本。

**5. 📚 核心考点:** S3 Object Lambda 在动态数据处理中的应用。

#### 📝 [334/529] Amazon DynamoDB Accelerator (DAX) 缓存

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** DynamoDB 读取密集型应用。
* **需求：** 降低读取延迟，减少读取成本。
* **访问模式：** 重复读取相同数据，热点数据。
* **性能要求：** 微秒级延迟，高吞吐量。

**2. ⚡ 秒杀思路**
* **DynamoDB 性能优化：**
    * **增加 RCUs:** 提高吞吐量但成本高。
    * **应用层缓存:** 开发复杂，一致性难保证。
    * **DAX:** 专用的 DynamoDB 缓存层。
* **DAX 核心特性：**
    * **内存缓存:** 将热点数据缓存到内存。
    * **自动失效:** 与 DynamoDB 自动同步。
    * **兼容性:** 与 DynamoDB API 完全兼容。
    * **高可用:** 多可用区部署。
* **性能优势：**
    * **选项 A (增加容量):** 成本高，延迟改善有限。
    * **选项 B (应用缓存):** 开发复杂，一致性风险。
    * **选项 C (DAX):** 
        * 显著降低读取延迟。
        * 减少 DynamoDB 读取成本。
        * 无缝集成。
    * **选项 D (ElastiCache):** 需要自己实现同步逻辑。
* **配置要点：**
    * 评估缓存命中率预期。
    * 选择合适的节点类型和数量。
    * 配置缓存策略和 TTL。
    * 监控缓存性能指标。

**3. ✅ 正确选项解析 (选项 C)**
* **DynamoDB DAX:** 专用的内存缓存服务，显著提升 DynamoDB 读取性能。

**5. 📚 核心考点:** DAX 在 DynamoDB 性能优化中的应用和配置最佳实践。

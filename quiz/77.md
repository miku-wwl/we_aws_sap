#### 📝 [77/529] VPC 传递路由陷阱 (TGW)

**1. 🕵️‍♂️ 题眼与约束分析**
* **拓扑：** On-prem <-> VPC A (Peering) <-> VPC B。
* **问题：** On-prem 无法访问 VPC B。
* **原因：** **VPC Peering 不支持传递路由**。

**2. ⚡ 秒杀思路**
* **打破限制：** 使用 **Transit Gateway (TGW)** 构建星型网络。
* **锁定 A。**

**3. ✅ 正确选项解析 (选项 A)**
* **Transit Gateway:** 它就像一个云上的核心路由器。将 VPN、VPC A、VPC B 都"插"到 TGW 上，它们之间就自动互通了（只要路由表配置正确）。它原生支持传递路由。

**4. ❌ 错误选项排查**
* **选项 B (VPN Mesh):**
    * *错误原因：* 方案描述很乱（VPN 连接 TGW 再连 Peering？）。且如果要在 On-prem 和 VPC B 之间建 VPN，不需要 TGW 也能建。但这增加了管理复杂度（Mesh 网络）。
* **选项 C (BGP 传播):**
    * *错误原因：* **技术限制**。无论怎么配 BGP 或路由表，Peering 的底层协议决定了数据包不能"跨越"中间的 VPC。A 的包到了 B，不能再转发给 C。
* **选项 D (Modify VGW):**
    * *错误原因：* VGW 是绑定在单个 VPC 上的，不能跨 VPC 共享或路由。

**5. 📚 核心考点:** VPC Peering 局限性与 TGW 的优势。

#### 📝 [273/529] CloudFront S3 静态网站安全加固

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** S3 静态网站 + CloudFront 分发。
* **问题：** 直接访问 S3 网站端点存在安全风险。
* **需求：** 强制所有流量通过 CloudFront。
* **当前：** S3 静态网站配置 + CloudFront OAI。

**2. ⚡ 秒杀思路**
* **S3 静态网站访问方式：**
    * **网站端点：** `http://bucket.s3-website-region.amazonaws.com`
    * **REST API 端点：** `http://bucket.s3.amazonaws.com`
* **安全控制：**
    * **选项 A (Block public access):** 必须启用，但不能阻止已配置的静态网站访问。
    * **选项 B (OAI):** 已有，但仅限制 REST API 访问，不影响网站端点。
    * **选项 C (CloudFront Origin Access):** 
        * 禁用 S3 静态网站托管。
        * 使用 S3 作为 **Origin** 而非网站端点。
        * 所有访问强制通过 CloudFront。
    * **选项 D (WAF):** 不能阻止直接 S3 访问。
* **选择：** C 是最安全的方案。

**3. ✅ 正确选项解析 (选项 C)**
* **Disable S3 Website Hosting:** 移除直接访问路径。
* **Use S3 as Origin:** 强制通过 CloudFront 访问。

**5. 📚 核心考点:** S3 访问控制与 CloudFront 安全架构设计。

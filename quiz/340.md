#### 📝 [340/529] AWS Cloud WAN 全球广域网管理

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 全球分支机构需要统一网络管理。
* **需求：** 集中式网络策略，简化的运维。
* **网络要求：** SD-WAN 功能，安全连接。
* **管理目标：** 统一管理全球网络基础设施。

**2. ⚡ 秒杀思路**
* **企业广域网方案：**
    * **传统 WAN:** 硬件设备，配置复杂。
    * **第三方 SD-WAN:** 集成成本高。
    * **AWS Cloud WAN:** 云原生 SD-WAN 解决方案。
* **Cloud WAN 核心特性：**
    * **集中管理:** 统一管理全球网络策略。
    * **自动化配置:** 简化网络设备配置。
    * **安全性:** 内置安全策略和加密。
    * **性能优化:** 智能路由和带宽管理。
* **网络优势：**
    * **选项 A (传统 WAN):** 管理复杂，成本高。
    * **选项 B (多个 TGWs):** 配置复杂，缺乏统一视图。
    * **选项 C (Cloud WAN):** 
        * 云原生 SD-WAN。
        * 集中式管理平台。
        * 自动化网络运维。
    * **选项 D (Direct Connect):** 仅限骨干连接，不管理分支。
* **架构设计：**
    * 创建全局网络和核心网络。
    * 定义网络策略和分段。
    * 连接分支站点和云资源。
    * 实施监控和性能优化。

**3. ✅ 正确选项解析 (选项 C)**
* **AWS Cloud WAN:** 云原生 SD-WAN 服务，简化全球企业网络管理。

**5. 📚 核心考点:** Cloud WAN 在企业网络现代化转型中的作用和优势。

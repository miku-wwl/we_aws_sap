#### 📝 [287/529] AWS Network Firewall 高级规则配置

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** VPC 内部流量监控和过滤。
* **需求：** 基于域名、协议、端口的精细控制。
* **约束：** 需要深度包检测，支持威胁情报集成。
* **合规要求：** 必须记录和审计所有流量。

**2. ⚡ 秒杀思路**
* **网络安全工具对比：**
    * **NACL:** 无状态，规则简单。
    * **Security Group:** 有状态但规则有限。
    * **WAF:** 专注于 Web 应用。
    * **Network Firewall:** 企业级防火墙功能。
* **Network Firewall 特性：**
    * **5-tuple 过滤:** 源/目标 IP、端口、协议。
    * **域名过滤:** 基于域名的访问控制。
    * **威胁情报:** 自动更新恶意 IP/域名列表。
    * **深度包检测:** IPS/IDS 功能。
* **配置策略：**
    * **选项 A (NACL):** 功能有限。
    * **选项 B (Security Group):** 不支持域名过滤。
    * **选项 C (Network Firewall):** 
        * 完整的防火墙功能。
        * 威胁情报集成。
        * 详细流量日志。
    * **选项 D (Gateway Endpoint):** 只控制特定服务访问。
* **选择：** Network Firewall 提供企业级网络安全防护。

**3. ✅ 正确选项解析 (选项 C)**
* **AWS Network Firewall:** VPC 级别的企业级防火墙，支持深度包检测。

**5. 📚 核心考点:** Network Firewall 规则配置与高级安全特性。

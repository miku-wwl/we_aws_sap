#### 📝 [236/529] 强制标签策略 (SCP + Tag Policy)

**1. 🕵️‍♂️ 题眼与约束分析**
* **需求：**
    1.  Standardize tags (标准化标签)。
    2.  Require specific value on creation (创建时必须有特定值)。
    3.  Each OU has unique tag value (每个 OU 值不同)。
* **工具：** SCP + Tag Policy。

**2. ⚡ 秒杀思路**
* **强制创建时带标签：**
    * 使用 **SCP (Service Control Policy)**。配置 `Deny` 动词，条件是 `Null` Tag 或 Tag Value 不匹配。
    * 既然每个 OU 的标签值不同，SCP 必须挂在 **OUs** 上，而不是 Root。
* **定义合规值：**
    * 使用 **Tag Policy**。它可以定义 Tag Key 的大小写和允许的 Values。
    * Tag Policy 也可以挂在 OUs 上。
* **选项对比：**
    * **选项 A:**
        * SCP: Deny creation without required tags. (Attached to OUs).
        * Tag Policy: Define values. (Attached to OUs).
        * 这是最符合"每个 OU 有独特值"的配置。
    * 选项 B: Tag Policy attached to Management Account? 这样所有 OU 继承同一个策略，如果不细分，很难实现"每个 OU 独特值"。通常策略下发到 OU 更精准。但其实 Tag Policy 支持继承和覆盖。
    * **关键是 SCP:**
        * SCP 必须在 OU 级别，因为它包含了具体的"允许值"逻辑（如果 SCP 仅仅是强制 Key 存在，那可以在 Root。但题目说 "Require specific value"）。如果每个 OU 值不同，SCP 逻辑就不同，所以 SCP 必须挂在 OU。
    * **结论：** 选项 A 的配置位置（OUs）最合理。
    * **锁定 A。**

**3. ✅ 正确选项解析 (选项 A)**
* **SCP:** 强制执行（Preventive）。
* **Tag Policy:** 定义标准（合规性）。
* **Attachment:** OU 级别实现差异化。

**5. 📚 核心考点:** 标签治理的组合拳 (SCP + Tag Policy)。

#### 📝 [358/529] AWS Security Hub 安全中心

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 需要统一的安全监控和合规管理。
* **需求：** 集中安全告警，合规检查。
* **约束：** 多账户，多区域部署。
* **安全要求：** 自动化安全评估，统一视图。

**2. ⚡ 秒杀思路**
* **安全管理挑战：**
    * **分散工具:** 多个安全工具，难以统一管理。
    * **手动合规:** 检查耗时，容易遗漏。
    * **告警泛滥:** 误报多，优先级不明确。
* **Security Hub 解决方案：**
    * **统一聚合:** 集中来自多个安全服务的告警。
    * **合规检查:** 自动化合规性评估。
    * **安全评分:** 整体安全态势评估。
    * **工作流自动化:** 告警处理和响应自动化。
* **安全优势：**
    * **选项 A (单点工具):** 视角有限，无法全局掌控。
    * **选项 B (自定义):** 开发复杂，维护困难。
    * **选项 C (Security Hub):** 
        * 企业级安全中心。
        * 自动化合规检查。
        * 统一安全视图。
    * **选项 D (无管理):** 安全风险高，合规困难。
* **配置要点：**
    * 启用安全标准和控制。
    * 集成第三方安全工具。
    * 配置告警规则和通知。
    * 实施自动化响应流程。

**3. ✅ 正确选项解析 (选项 C)**
* **AWS Security Hub:** 企业级安全管理中心，提供统一的安全视图和合规管理。

**5. 📚 核心考点:** Security Hub 在企业安全治理中的作用和配置方法。

#### 📝 [277/529] S3 Multi-Region Access Point 配置

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 全球应用，需要在多个区域访问 S3 数据。
* **需求：** 降低延迟，提高可用性。
* **约束：** 数据需要在不同区域间复制。

**2. ⚡ 秒杀思路**
* **S3 多区域访问模式：**
    * **S3 Cross-Region Replication (CRR):** 异步复制，需要配置存储桶策略。
    * **S3 Multi-Region Access Point (MRAP):** 
        * 单一全球端点。
        * 自动路由到最近的区域。
        * 简化访问管理。
* **配置步骤：**
    * **选项 A:** 仅 CRR，无统一端点。
    * **选项 B:** 仅 MRAP，无数据复制。
    * **选项 C:** CRR + MRAP，完整解决方案。
    * **选项 D:** Route 53，增加复杂度。
* **架构设计：** MRAP 需要 CRR 支持才能提供真正的多区域访问。

**3. ✅ 正确选项解析 (选项 C)**
* **CRR + MRAP:** 实现真正的全球低延迟、高可用性访问。

**5. 📚 核心考点:** S3 Multi-Region Access Point 与跨区域复制的组合使用。

#### 📝 [376/529] AWS Cognito 用户身份池

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 需要管理应用用户身份认证。
* **需求：** 用户注册登录，社交登录集成。
* **约束：** 多平台支持，可扩展性。
* **安全要求：** MFA 支持，安全合规。

**2. ⚡ 秒杀思路**
* **用户认证方案：**
    * **自建认证:** 开发复杂，安全风险高。
    * **第三方服务:** 集成成本高，数据安全风险。
    * **Cognito:** AWS 托管用户认证服务。
* **Cognito 特性：**
    * **用户池:** 完整的用户注册登录管理。
    * **身份池:** 临时 AWS 凭证获取。
    * **社交集成:** 支持 Google、Facebook 等社交登录。
    * **MFA 支持:** 多因素认证保护。
* **认证优势：**
    * **选项 A (自定义):** 开发成本高，安全难保证。
    * **选项 B (OAuth 2.0):** 需要自己实现用户管理。
    * **选项 C (Cognito):** 
        * 托管用户认证。
        * 社交登录集成。
        * 企业级安全保障。
    * **选项 D (无认证):** 安全风险高。

**3. ✅ 正确选项解析 (选项 C)**
* **Cognito:** 企业级用户认证服务，提供完整的身份管理解决方案。

**5. 📚 核心考点:** Cognito 在应用认证中的配置和最佳实践。

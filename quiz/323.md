#### 📝 [323/529] AWS Cloud Map 服务发现

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 微服务架构需要动态服务发现。
* **需求：** 自动服务注册，健康检查。
* **约束：** 服务实例动态变化，需要实时更新。
* **集成要求：** 与 ECS、EKS 原生集成。

**2. ⚡ 秒杀思路**
* **服务发现方案：**
    * **DNS 服务:** 静态配置，无法动态更新。
    * **配置中心:** 需要客户端轮询更新。
    * **Cloud Map:** 专门的云原生服务发现。
* **Cloud Map 核心特性：**
    * **服务注册:** 自动注册和注销服务实例。
    * **健康检查:** 自动检测服务健康状态。
    * **DNS API:** 支持 DNS 和 HTTP API 查询。
    * **命名空间:** 逻辑隔离服务目录。
* **发现优势：**
    * **选项 A (Route 53):** 主要用于外部服务，内网复杂。
    * **选项 B (配置文件):** 静态配置，无法自动更新。
    * **选项 C (Cloud Map):** 
        * 动态服务发现。
        * 自动健康检查。
        * 云服务原生集成。
    * **选项 D (ELB):** 负载均衡器，非服务发现。
* **使用场景：**
    * ECS/EKS 服务实例注册。
    * Lambda 函数服务发现。
    * 跨区域服务路由。

**3. ✅ 正确选项解析 (选项 C)**
* **AWS Cloud Map:** 云原生服务发现和注册中心，简化微服务架构。

**5. 📚 核心考点:** Cloud Map 在微服务架构中的作用和服务发现最佳实践。

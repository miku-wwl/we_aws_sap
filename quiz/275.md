#### 📝 [275/529] ECS 任务角色 vs 任务执行角色

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** ECS 任务需要访问 S3 和 DynamoDB。
* **安全：** 遵循最小权限原则。
* **需求：** 为容器配置适当的 IAM 权限。

**2. ⚡ 秒杀思路**
* **ECS IAM 角色类型：**
    * **Task Execution Role:** 
        * 用于 ECS 服务拉取镜像、写日志。
        * 由 ECS 服务使用。
    * **Task Role:** 
        * 容器内应用代码使用的角色。
        * 访问 AWS 服务（S3、DynamoDB 等）。
* **权限分配：**
    * **选项 A:** 任务角色访问 AWS 服务 ✓
    * **选项 B:** 执行角色访问 AWS 服务 ✗
    * **选项 C:** 两者混合 ✓
    * **选项 D:** EC2 实例角色 ✗
* **最佳实践：** 分离执行角色（基础设施）和任务角色（应用）。

**3. ✅ 正确选项解析 (选项 C)**
* **Task Role:** 应用访问 AWS 服务权限。
* **Task Execution Role:** ECS 基础设施权限。

**5. 📚 核心考点:** ECS IAM 角色分离原则与安全最佳实践。

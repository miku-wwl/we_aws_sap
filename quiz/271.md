#### 📝 [271/529] API Gateway 直连 SQS 架构优化

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 移动应用通过 API Gateway 接收用户数据。
* **后端：** SQS 队列 + EC2 消费者处理。
* **当前架构：** API Gateway → Lambda → SQS → EC2。
* **痛点：** Lambda 作为简单代理，增加成本和复杂度。

**2. ⚡ 秒杀思路**
* **架构简化原则：** 减少中间层，降低运维成本。
* **API Gateway 集成能力：**
    * **AWS Service Integration:** 直接调用 AWS 服务 API。
    * **SQS 集成:** 支持 `SendMessage` 操作。
    * **IAM 权限:** 自动处理凭证。
* **实施方案：**
    * **选项 A (保持 Lambda):** 无架构优化。
    * **选项 B (直接集成 SQS):** 
        * 配置 API Gateway AWS 集成。
        * 目标服务：SQS `SendMessage`。
        * 移除 Lambda 层。
    * **选项 C (使用 Kinesis):** 不符合现有 SQS 架构。
    * **选项 D (使用 Step Functions):** 增加复杂度。
* **选择：** 直接集成是最佳简化方案。

**3. ✅ 正确选项解析 (选项 B)**
* **API Gateway SQS Integration:** 消除中间层，降低成本，简化架构。

**5. 📚 核心考点:** API Gateway 服务集成模式与架构简化最佳实践。

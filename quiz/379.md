#### 📝 [379/529] AWS Control Tower 控制塔

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 需要建立和管理多账户环境。
* **需求：** 自动化部署，治理策略。
* **约束：** 大规模账户，合规要求。
* **治理目标：** 统一管控，防護措施。

**2. ⚡ 秒杀思路**
* **多账户治理方案：**
    * **手动创建:** 效率低，配置不一致。
    * **自定义脚本:** 维护困难。
    * **Control Tower:** 托管多账户治理。
* **Control Tower 核心功能：**
    * **Landing Zone:** 标准化账户架构。
    * **Guardrails:** 自动化治理策略。
    * **Account Factory:** 标准化账户创建。
    * **集中管控:** 统一策略管理。
* **治理优势：**
    * **选项 A (Organizations):** 需要大量自定义。
    * **选项 B (手动):** 配置不一致，难以维护。
    * **选项 C (Control Tower):** 
        * 企业级多账户治理。
        * 自动化部署和管理。
        * 内置最佳实践。
    * **选项 D (无治理):** 安全风险，合规问题。

**3. ✅ 正确选项解析 (选项 C)**
* **Control Tower:** 企业级多账户治理服务，简化 Landing Zone 部署和管理。

**5. 📚 核心考点:** Control Tower 在企业多账户治理中的配置和最佳实践。

#### 📝 [313/529] AWS Amplify 全栈应用开发

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 前端开发者需要快速构建全栈应用。
* **需求：** 前端 + 后端 + 托管一体化解决方案。
* **约束：** 尽量减少后端代码编写。
* **功能要求：** 用户认证、数据存储、API 接口。

**2. ⚡ 秒杀思路**
* **全栈开发平台对比：**
    * **传统方式:** 前端 + 独立后端服务，开发复杂。
    * **Serverless Framework:** 需要 YAML 配置，学习曲线。
    * **Amplify:** 声明式全栈开发框架。
* **Amplify 核心组件：**
    * **CLI 工具:** 命令行界面管理后端资源。
    * **UI 库:** 预构建 UI 组件。
    * **托管服务:** 自动 CI/CD 和托管。
    * **类别支持:** Auth、Storage、API、Function 等。
* **开发优势：**
    * **选项 A (传统):** 开发周期长，运维复杂。
    * **选项 B (自定义):** 需要全栈技能。
    * **选项 C (Amplify):** 
        * 简化全栈开发。
        - 自动化部署。
        * 丰富的预构建组件。
    * **选项 D (App Runner):** 仅限应用部署，不提供后端服务。
* **使用流程：**
    * 初始化 Amplify 项目。
    * 添加需要的类别（Auth、API 等）。
    * 连接前端应用。
    * 推送代码自动部署。

**3. ✅ 正确选项解析 (选项 C)**
* **AWS Amplify:** 面向前端开发者的全栈应用开发平台，简化云服务集成。

**5. 📚 核心考点:** Amplify 在现代 Web 应用开发中的优势和最佳实践。

#### 📝 [73/529] IoT 设备认证 (MQTT + X.509)

**1. 🕵️‍♂️ 题眼与约束分析**
* **规模：** 数千设备。
* **协议：** MQTT。
* **认证：** X.509 证书。
* **约束：** **Least operational overhead**。

**2. ⚡ 秒杀思路**
* **IoT 托管服务：** **AWS IoT Core** 原生支持题目所有要求。
* **锁定 C。**

**3. ✅ 正确选项解析 (选项 C)**
* **AWS IoT Core:** 是全托管的 MQTT 代理。它自带证书颁发机构 (CA) 和注册表，可以轻松管理数百万个设备的 X.509 证书和连接，运维开销几乎为零。

**4. ❌ 错误选项排查**
* **选项 A (Amazon MQ):**
    * *错误原因：* Amazon MQ 是基于 ActiveMQ/RabbitMQ 的。虽然支持 MQTT，但它主要是为后端服务设计的，**并发连接数有限**。为每个设备建一个队列更是不可行的架构。
* **选项 B (NLB + EC2 MQTT Broker):**
    * *错误原因：* **运维噩梦**。你需要自己维护 MQTT 软件（如 Mosquitto, EMQ X）的高可用、打补丁、扩容，还要自己实现 X.509 验证逻辑。
* **选项 D (API Gateway + NLB):**
    * *错误原因：* API Gateway 是 HTTP/WebSocket 的，虽然支持双向 TLS，但为了跑 MQTT 再去搞这一套转发非常繁琐且昂贵。

**5. 📚 核心考点:** AWS IoT Core 的核心能力。

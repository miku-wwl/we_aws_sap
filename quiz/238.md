#### 📝 [238/529] 集中备份与合规 (AWS Backup)

**1. 🕵️‍♂️ 题眼与约束分析**
* **资源：** EC2, EFS, RDS。
* **需求：**
    1.  Custom retention (自定义保留)。
    2.  **Copy to another Region** (跨区复制)。
    3.  **Single source of backup status** (统一视图)。
    4.  **Notification on failure** (失败通知)。
* **约束：** **Least operational overhead**。
* **选择：** 3 个步骤。

**2. ⚡ 秒杀思路**
* **核心服务：** **AWS Backup**。它能统一管理 EC2, EFS, RDS 的备份。
* **步骤分解：**
    1.  **创建计划：** Create Backup Plan with rules (频率、保留)。 $→$ **选中 A**。
    2.  **配置复制：** 在 Backup Plan 中配置 **Cross-Region Copy**。 $→$ **选中 B**。
    3.  **配置通知：** AWS Backup 支持 **Backup Vault Notifications** (基于 SNS)。可以配置当 Job 状态不为 COMPLETED 时发送通知。
        * 选项 D: "Add SNS topic to backup plan..."? 实际上是配置在 **Backup Vault** 上。但选项 D 描述了 SNS 发送通知的意图。
        * 选项 C: Lambda 复制？AWS Backup 原生支持复制，不需要 Lambda。
    * $→$ **选中 D**。
* **排除法：**
    * E (DLM): DLM 主要管 EBS，管不了 RDS/EFS。
    * F (RDS Snapshot): 分散管理，不符合 "Single source"。
* **锁定 A, B, D。**

**3. ✅ 正确选项解析 (选项 A, B, D)**
* **AWS Backup Plan:** 定义策略（备份+复制）。
* **SNS Notification:** 监控任务状态。

**5. 📚 核心考点:** AWS Backup 的全功能应用 (Plan, Copy, Notify)。

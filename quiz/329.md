#### 📝 [329/529] Amazon Quantum Ledger Database (QLDB)

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 需要不可篡改的交易记录系统。
* **需求：** 数据完整性和透明性，可审计。
* **约束：** 需要证明数据未被修改。
* **应用场景：** 金融交易、供应链、身份管理。

**2. ⚡ 秒杀思路**
* **不可变数据库对比：**
    * **传统数据库:** 可以修改历史数据。
    * **区块链技术:** 复杂度高，性能差。
    * **QLDB:** 区块链特性的关系数据库。
* **QLDB 核心特性：**
    * **不可变账本:** 所有修改都有加密证明。
    * **可验证历史:** 完整的数据修改历史追踪。
    * **ACID 事务:** 关系数据库的事务特性。
    * **SQL 兼容:** 标准 SQL 查询语言。
* **技术优势：**
    * **选项 A (RDS):** 数据可被修改，无法保证完整性。
    * **选项 B (DynamoDB):** 版本控制复杂，审计困难。
    * **选项 C (QLDB):** 
        * 原生不可变设计。
        * 加密证明的数据完整性。
        * 简单的 SQL 接口。
    * **选项 D (区块链):** 复杂度高，性能和扩展性问题。
* **使用案例：**
    * 银行交易记录。
    * 供应链溯源。
    * 身份变更历史。

**3. ✅ 正确选项解析 (选项 C)**
* **Amazon QLDB:** 具有区块链特性的不可变数据库，提供可验证的数据完整性。

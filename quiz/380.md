#### 📝 [380/529] AWS Organizations 组织管理

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 需要集中管理多个 AWS 账户。
* **需求：** 统一账单，权限控制。
* **约束：** 大规模环境，合规要求。
* **管理目标：** 集中策略，成本优化。

**2. ⚡ 秒杀思路**
* **账户管理方案：**
    * **分散管理:** 效率低，成本高。
    * **第三方工具:** 集成复杂。
    * **Organizations:** AWS 原生组织管理。
* **Organizations 特性：**
    * **集中账单:** 统一账单管理。
    * **SCP 策略:** 服务控制策略。
    * **账户分组:** 逻辑组织结构。
    * **权限边界:** 账户级权限限制。
* **管理优势：**
    * **选项 A (单独管理):** 效率低，难以控制。
    * **选项 B (脚本):** 维护困难。
    * **选项 C (Organizations):** 
        * 原生账户管理。
        * 集中策略控制。
        * 成本优化能力。
    * **选项 D (无管理):** 管理混乱，成本高。

**3. ✅ 正确选项解析 (选项 C)**
* **Organizations:** 原生多账户管理服务，提供集中控制和成本管理。

**5. 📚 核心考点:** Organizations 在企业多账户环境中的配置和管理。

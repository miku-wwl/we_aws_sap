#### 📝 [239/529] 基因数据实时流处理 (Kinesis Streams + EMR)

**1. 🕵️‍♂️ 题眼与约束分析**
* **输入：** 8 KB/sec per device (小数据流)。
* **需求：**
    1.  **Near-real-time analysis** (近实时分析)。
    2.  Flexible, parallel, durable.
    3.  Deliver to Data Warehouse (Redshift).
* **分析：** 需要强大的计算能力（基因数据）。

**2. ⚡ 秒杀思路**
* **摄取层：**
    * **Kinesis Data Streams (KDS) (B):** 支持实时流摄取，保留数据（Durable），支持并行消费。
    * Firehose (A): 主要用于加载数据（Load），分析能力弱（只能简单的 Lambda 转换）。题目要求 "Analyze data... using Kinesis Client"，通常指自定义应用或 EMR/Spark Streaming。
    * API Gateway (D): 虽然可以作为前端，但后端还需要流。
* **分析与存储层：**
    * **EMR (Spark/Flink):** 可以直接消费 Kinesis Streams，进行复杂的基因数据实时分析。
    * **Redshift:** 数仓存储。
    * **选项 B:** KDS -> Kinesis Client (on EMR?) -> Redshift。这是符合"近实时、并行分析"的高性能架构。
    * **选项 A:** Firehose -> RDS? RDS 不适合分析海量基因数据。
    * **选项 C:** S3 -> SQS -> Kinesis? 乱序。
* **锁定 B。**

**3. ✅ 正确选项解析 (选项 B)**
* **Kinesis Data Streams:** 实时数据总线。
* **EMR:** 大数据实时处理引擎。

**5. 📚 核心考点:** 实时流处理架构 (Kinesis + EMR)。

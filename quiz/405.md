#### 📝 [405/529] EKS 节点弹性与分布 (Topology Spread)

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** EKS, Stateless Pods, Unpredictable number (不可预测)。
* **特点：** Short period creation (短时大量创建)。
* **需求：** **Maximize node resilience** (最大化节点弹性/可用性)。

**2. ⚡ 秒杀思路**
* **弹性策略：**
    * 这里的 "Resilience" 通常指高可用，即避免所有 Pod 都在一个节点或一个 AZ。
    * **Topology Spread Constraints (D):** Kubernetes 原生功能，可以控制 Pod 在不同拓扑域（如 AZ、节点）上的分布。配置 `topologyKey: topology.kubernetes.io/zone` 可以强制 Pod 跨 AZ 分布。这是最大化弹性的正确姿势。
* **其他选项：**
    * A: EKS Control Plane 是 AWS 托管的，用户无法控制其部署位置（它本身就是多 AZ 的）。
    * B: Less node groups? 这跟弹性没关系。
    * C: Cluster Autoscaler over-provisioning (过度配置)? 这提高了启动速度（减少 Pending 时间），但没解决 Pod 分布的弹性问题（可能都挤在一个 AZ）。
    * **锁定 D。**

**3. ✅ 正确选项解析 (选项 D)**
* **Topology Spread Constraints:** K8s 原生调度策略，确保 Pod 跨故障域（AZ）均匀分布。

**5. 📚 核心考点:** Kubernetes (EKS) 的高可用调度配置。

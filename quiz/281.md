#### 📝 [281/529] RDS Custom for Oracle 企业级部署

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** Oracle E-Business Suite 部署需求。
* **特殊要求：** 
    * 需要访问底层操作系统。
    * 安装特定 Oracle 补丁和配置。
    * 遵循 Oracle 支持政策。
* **标准 RDS 限制：** 无法满足 OS 访问需求。

**2. ⚡ 秒杀思路**
* **RDS 版本选择：**
    * **Amazon RDS:** 完全托管，无 OS 访问权限。
    * **RDS Custom for Oracle:** 
        * 提供有限的 OS 访问权限。
        * 支持安装特定 Oracle 组件。
        * 保持托管服务的便利性。
    * **EC2 + Oracle:** 完全自管理，运维负担重。
* **关键决策因素：**
    * **Oracle E-Business Suite:** 需要 OS 级别的配置。
    * **补丁管理:** 需要安装 vendor 提供的补丁。
    * **性能调优:** 可能需要 OS 级别的参数调整。
* **选项分析：**
    * **选项 A (标准 RDS):** 权限不足，无法满足需求。
    * **选项 B (EC2):** 自管理，违反简化运维目标。
    * **选项 C (RDS Custom):** 完美平衡托管服务和控制权。
    * **选项 D (RDS on Outposts):** 本地部署，不解决权限问题。
* **锁定 C。**

**3. ✅ 正确选项解析 (选项 C)**
* **RDS Custom:** 为需要特殊配置的企业级数据库场景提供定制化控制。

**5. 📚 核心考点:** RDS Custom 的使用场景和企业级数据库部署策略。

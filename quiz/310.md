#### 📝 [310/529] AWS Fault Injection Simulator 混沌工程

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 需要测试系统故障恢复能力。
* **需求：** 受控故障注入，验证高可用性。
* **约束：** 不影响生产环境，可重复测试。
* **验证目标：** 自动故障转移，服务降级。

**2. ⚡ 秒杀思路**
* **故障测试方法：**
    * **手动故障:** 不可控，影响范围大。
    * **自定义脚本:** 开发复杂，覆盖面有限。
    * **FIS 服务:** 专业的混沌工程平台。
* **FIS 核心能力：**
    * **故障注入:** CPU、内存、网络、API 故障。
    * **目标选择:** 基于标签的资源选择。
    * **安全控制:** 停止机制、权限控制。
    * **集成监控:** 结合 CloudWatch 自动停止。
* **混沌工程价值：**
    * **选项 A (无测试):** 系统脆弱性未知。
    * **选项 B (手动):** 风险高，不可重复。
    * **选项 C (FIS):** 
        * 受控故障注入。
        * 自动化实验执行。
        * 安全可靠的测试。
    * **选项 D (第三方):** 集成复杂，成本高。
* **实验设计：**
    * 定义实验模板。
    * 选择目标和故障类型。
    * 配置停止条件和监控。
    * 执行实验并分析结果。

**3. ✅ 正确选项解析 (选项 C)**
* **AWS FIS:** 企业级混沌工程平台，提升系统韧性和可靠性。

**5. 📚 核心考点:** FIS 在系统可靠性验证中的应用和混沌工程最佳实践。

#### 📝 [78/529] 旧 SMTP 迁移到 SES (凭证与 TLS)

**1. 🕵️‍♂️ 题眼与约束分析**
* **限制：** 旧 SMTP Server 停用，App 只能用 SMTP，不支持 TLS，用端口 25。
* **目标：** 连接 SES (要求 STARTTLS)。

**2. ⚡ 秒杀思路**
* **核心矛盾：** App 不支持 TLS vs SES 强制 TLS。
* **凭证陷阱：** SMTP 接口必须用 **SMTP Credentials**，不能用 IAM Role。
* **锁定 B。** *（注：这里有一个逻辑修正，详见下方排错）*

**3. ✅ 正确选项解析 (选项 B)**
* **SMTP Credentials:** 这是连接 SES SMTP Endpoint 的唯一钥匙。
* **STARTTLS:** 这是 SES SMTP 的硬性要求。
* *关于 B 的合理性讨论：* 虽然题目说"Old SMTP server doesn't support TLS"，但这不代表 App 无法配置。如果 App 确实无法配置，通常需要一个本地代理（如 A 所述）。但 A 错在"IAM Role"。在 AWS 考试中，**"SMTP 必须用 SMTP Credentials"** 这个考点的优先级极高。因此，即使 B 假设 App 可以修改配置，也是唯一"凭证逻辑"正确的选项。

**4. ❌ 错误选项排查**
* **选项 A (TLS Wrapper + IAM Role):**
    * *错误原因：* **鉴权方式错误**。SES 的 SMTP 接口**不识别** EC2 绑定的 IAM Role。你必须在 SMTP 握手时提供用户名和密码（AUTH LOGIN）。Wrapper 只是解决加密，没解决认证。
* **选项 C/D (API/SDK):**
    * *错误原因：* 违反题目约束 "Application can **only** use SMTP"。

**5. 📚 核心考点:** SES SMTP 接口的特殊性 (Auth & TLS)。

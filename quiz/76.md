#### 📝 [76/529] RDS 跨区域高可用 (Read Replica)

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 跨区域高可用。
* **数据库：** RDS for MySQL。

**2. ⚡ 秒杀思路**
* **RDS MySQL DR:** 只有 **Cross-Region Read Replica** 是标准方案。
* **锁定 D。**

**3. ✅ 正确选项解析 (选项 D)**
* **Promote Read Replica:** 在灾难发生时，将从属区域的只读副本提升为独立的读写实例，然后将应用指向它。这是 RDS（非 Aurora）的标准 DR 流程。

**4. ❌ 错误选项排查**
* **选项 A (Automated Backups):**
    * *错误原因：* **RTO 太长**。跨区域恢复快照需要将数据从 S3 拷贝并启动新实例，可能需要几十分钟到数小时，达不到 "Highest Availability" 标准。
* **选项 B/C (Global Tables / Lambda):**
    * *错误原因：* **术语错误/反模式**。"Global Tables" 是 DynamoDB 的功能，RDS MySQL 没有这个叫法（Aurora 叫 Global Database）。用 Lambda 自己写代码复制数据（B/C）是极其不可靠的，容易导致数据不一致。

**5. 📚 核心考点:** RDS for MySQL 的原生灾备能力 (RR)。

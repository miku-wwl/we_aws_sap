#### 📝 [335/529] AWS Global Accelerator 全球网络优化

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 全球用户访问应用，需要优化网络性能。
* **需求：** 降低延迟，提高可用性。
* **网络特征：** 跨地域访问，TCP/UDP 流量。
* **性能目标：** 稳定的网络性能，自动故障转移。

**2. ⚡ 秒杀思路**
* **全球网络优化方案：**
    * **Route 53:** DNS 级别的负载均衡。
    * **CloudFront:** 仅限 HTTP/HTTPS 流量。
    * **Global Accelerator:** 网络层流量优化。
* **Global Accelerator 特性：**
    * **静态 IP:** 固定的全球 Anycast IP。
    * **AWS 全球网络:** 使用 AWS 优化网络骨干网。
    * **健康检查:** 自动故障检测和转移。
    * **流量转向:** 智能路由到最优终端节点。
* **网络优势：**
    * **选项 A (Route 53):** DNS 层面，只能做简单路由。
    * **选项 B (CloudFront):** 仅限缓存内容，不支持动态流量。
    * **选项 C (Global Accelerator):** 
        * 网络层优化。
        * 支持所有 TCP/UDP 流量。
        * 全球网络骨干网。
    * **选项 D (Direct Connect):** 成本高，需要专用连接。
* **使用场景：**
    * 游戏服务器连接。
    * IoT 设备通信。
    * 全球化应用接入。

**3. ✅ 正确选项解析 (选项 C)**
* **AWS Global Accelerator:** 网络层流量优化服务，提供全球低延迟连接。

**5. 📚 核心考点:** Global Accelerator 在全球应用架构中的网络优化作用。

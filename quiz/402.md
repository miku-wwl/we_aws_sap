#### 📝 [402/529] 远程办公与多系统桌面 (WorkSpaces)

**1. 🕵️‍♂️ 题眼与约束分析**
* **用户：** 400 员工，混合 OS (Windows/Linux)。
* **需求：**
    1.  Remote work environment (远程办公)。
    2.  Integrate with On-prem AD (AD 集成)。
    3.  MFA。
    4.  **Replicate existing desktop experience** (复制桌面体验 -> 全功能桌面)。
* **工具：** WorkSpaces vs AppStream。

**2. ⚡ 秒杀思路**
* **WorkSpaces (A/C):** 提供持久化的、完整的云桌面（DaaS）。支持 Windows 和 Linux。非常适合替代员工的主办公电脑。
* **AppStream 2.0 (B/D):** 提供应用程序流（非持久化桌面）。虽然有 Desktop View，但主要用于分发特定应用，数据不持久化（除非配置 Home Folders）。题目要求 "Replicate existing desktop experience"，通常 WorkSpaces 更贴切。
* **AD 集成与 MFA:**
    * **AD Connector:** 连接本地 AD。
    * **MFA:** WorkSpaces 支持通过 **RADIUS** 服务器集成 MFA。
    * 选项 A 说 "Activate MFA using AWS Console"。AWS Console 不能直接给 WorkSpaces 开 MFA，必须配置 RADIUS。
    * 选项 C 说 "Configure RADIUS server for MFA"。这是正确的配置方式。
* **锁定 C。**

**3. ✅ 正确选项解析 (选项 C)**
* **WorkSpaces:** 全功能云桌面。
* **AD Connector + RADIUS:** 实现 AD 集成与 MFA。

**5. 📚 核心考点:** WorkSpaces 的 MFA 配置依赖 (RADIUS)。

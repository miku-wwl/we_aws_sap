#### 📝 [373/529] AWS Secrets Manager 密码管理

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 需要安全存储和管理应用密码。
* **需求：** 自动密码轮换，集中管理。
* **约束：** 多应用集成，审计要求。
* **安全要求：** 加密存储，访问控制。

**2. ⚡ 秒杀思路**
* **密码管理挑战：**
    * **配置文件:** 明文存储，安全风险高。
    * **环境变量:** 不支持轮换，管理困难。
    * **Secrets Manager:** 专业的密码管理服务。
* **Secrets Manager 特性：**
    * **自动轮换:** 自动密码轮换支持。
    * **加密存储:** KMS 加密保护。
    * **多服务集成:** 与 RDS、Redshift 等集成。
    * **审计日志:** 完整的访问审计。
* **管理优势：**
    * **选项 A (配置文件):** 安全风险高，无轮换。
    * **选项 B (环境变量):** 不支持轮换，管理复杂。
    * **选项 C (Secrets Manager):** 
        * 自动密码轮换。
        * 安全加密存储。
        * 多服务集成。
    * **选项 D (明文):** 严重安全风险。
* **配置要点：**
    * 创建和存储密钥。
    * 配置自动轮换策略。
    * 设置访问权限和策略。
    * 监控密钥使用和轮换。

**3. ✅ 正确选项解析 (选项 C)**
* **Secrets Manager:** 专业的密码管理服务，提供自动轮换和安全存储。

**5. 📚 核心考点:** Secrets Manager 在应用安全管理中的配置和应用。

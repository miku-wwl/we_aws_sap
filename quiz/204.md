#### 📝 [204/529] 跨区域 EC2 备份与恢复 (AWS Backup)

**1. 🕵️‍♂️ 题眼与约束分析**
* **资产：** 20 EC2 (Persistent, Multiple EBS)。
* **RPO:** 1 Day (数据丢失 < 1天)。
* **RTO:** 1 Day (恢复 < 1天)。
* **资源：** Limited staff (人少) -> **Optimize operational efficiency**。
* **现有：** CloudFormation for network。

**2. ⚡ 秒杀思路**
* **备份工具：**
    * **SSM Automation (A):** 手写脚本管理快照？运维太重。
    * **DLM (B):** Data Lifecycle Manager 可以管理快照，但恢复过程比较手动（需要从快照创建卷，再挂载，或者创建 AMI）。不如 AWS Backup 集成度高。
    * **DataSync (D):** 它是文件级同步，不适合 EC2 整机/EBS 块级备份。
    * **AWS Backup (C):**
        * **跨区域复制:** 原生支持将备份复制到另一个区域。
        * **恢复:** 支持 **Restore entire instance** (恢复整个实例)，自动处理 AMI、EBS 挂载等配置。
        * **管理:** "Backup Plan" 策略化管理，最省心。
    * $\\rightarrow$ **选中 C**。
* **锁定 C。**

**3. ✅ 正确选项解析 (选项 C)**
* **AWS Backup:** 集中管理跨区域备份和恢复，极大降低运维负担。

**5. 📚 核心考点:** AWS Backup 的跨区域复制与整机恢复能力。

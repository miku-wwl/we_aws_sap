#### 📝 [309/529] Amazon Managed Grafana 可视化监控

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 需要统一的运维监控仪表板。
* **需求：** 多数据源集成，团队协作。
* **约束：** 需要企业级安全和权限管理。
* **运维要求：** 减少维护负担，自动更新。

**2. ⚡ 秒杀思路**
* **可视化平台对比：**
    * **自建 Grafana:** 需要自己管理服务器和插件。
    * **CloudWatch Dashboard:** 仅限 AWS 数据源。
    * **QuickSight:** 侧重商业智能，非运维监控。
    * **Managed Grafana:** 托管式 Grafana 服务。
* **Managed Grafana 优势：**
    * **无服务器:** 无需管理底层基础设施。
    * **AWS 集成:** 原生支持 20+ AWS 服务。
    * **SSO 集成:** 支持 SAML 和 IAM Identity Center。
    * **自动更新:** 平台和插件自动更新。
* **功能特性：**
    * **选项 A (自建):** 运维负担重，安全配置复杂。
    * **选项 B (CloudWatch):** 数据源受限，功能单一。
    * **选项 C (Managed Grafana):** 
        * 托管运维体验。
        - 丰富的数据源支持。
        * 企业级安全集成。
    * **选项 D (第三方):** 成本高，集成复杂。
* **配置要点：**
    * 创建工作空间。
    * 配置数据源连接。
    * 设置用户和团队权限。
    * 导入或创建仪表板。

**3. ✅ 正确选项解析 (选项 C)**
* **Amazon Managed Grafana:** 企业级运维可视化平台，简化监控仪表板管理。

**5. 📚 核心考点:** Managed Grafana 的优势及在运维监控体系中的应用。

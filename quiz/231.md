#### 📝 [231/529] 本地 SQL Server 备份到云 (FSx vs Gateway)

**1. 🕵️‍♂️ 题眼与约束分析**
* **源端：** 本地 SQL Server。
* **数据：** 200 GB/night Export (每日导出)。
* **连接：** 10 Gbps DX。
* **目标：** S3 (Cloud Storage)。
* **需求：** **Most cost-effective** (最具成本效益)。

**2. ⚡ 秒杀思路**
* **方案 A (File Gateway):**
    * 文件网关提供 SMB 共享，后端直接是 **S3**。
    * 数据库导出写入 SMB 共享 $→$ 自动上传到 S3。
    * **成本：** S3 标准存储费用 + 网关实例费用（很低）。这是最便宜的长期存储方案。
* **方案 B/C (FSx for Windows):**
    * FSx 是托管的文件系统，虽然也能做 SMB，但主要是为了运行 Windows 应用。
    * 用 FSx 存备份？太贵了！FSx 存储单价比 S3 贵很多。且 FSx 通常用于热数据，不是归档备份。
* **方案 D (Volume Gateway):**
    * 卷网关提供 iSCSI 块存储。虽然也可以，但不如文件网关的 SMB 共享直观（SQL Server 直接支持写 UNC 路径）。且卷网关主要用于快照备份，管理起来比文件网关的文件级访问麻烦。
* **结论：** A 选项利用 S3 作为最终存储，通过 File Gateway 提供 SMB 接口，完美适配 SQL Server 备份场景，且 S3 成本最低。
* **锁定 A。**

**3. ✅ 正确选项解析 (选项 A)**
* **Storage Gateway (File Gateway):** SMB 接口 + S3 后端，备份归档的神器。

**5. 📚 核心考点:** 混合云备份存储选型 (S3 File Gateway vs FSx)。

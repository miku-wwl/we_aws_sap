#### 📝 [320/529] AWS Application Composer 架构设计

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 需要设计 Serverless 应用架构。
* **需求：** 可视化架构设计，自动生成模板。
* **约束：** 减少 YAML 编写，降低学习曲线。
* **协作要求：** 团队共享和版本控制。

**2. ⚡ 秒杀思路**
* **基础设施即代码挑战：**
    * **手写 YAML:** 语法复杂，容易出错。
    * **CDK 编程:** 需要编程技能。
    * **可视化设计:** 降低门槛，提高效率。
* **Application Composer 特性：**
    * **拖拽设计:** 可视化组件组装。
    * **实时预览:** 架构图实时更新。
    * **模板生成:** 自动生成 CloudFormation 模板。
    * **组件库:** 丰富的预构建组件。
* **设计优势：**
    * **选项 A (手写):** 学习曲线陡峭，效率低。
    * **选项 B (CDK):** 需要编程背景。
    * **选项 C (Application Composer):** 
        * 可视化设计体验。
        * 自动模板生成。
        * 降低入门门槛。
    * **选项 D (第三方):** 集成成本高，更新不及时。
* **使用流程：**
    * 创建新的应用项目。
    * 从组件库拖拽服务到画布。
    * 配置组件属性和连接。
    * 导出生成的 CloudFormation 模板。

**3. ✅ 正确选项解析 (选项 C)**
* **AWS Application Composer:** 可视化 Serverless 应用设计工具，简化基础设施即代码。

**5. 📚 核心考点:** Application Composer 在 Serverless 架构设计中的优势和最佳实践。

#### 📝 [360/529] AWS Config Rules 配置合规

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 需要确保 AWS 资源配置符合企业标准。
* **需求：** 持续配置监控，自动合规检查。
* **约束：** 复杂的合规规则，动态资源变化。
* **治理要求：** 配置漂移检测，自动修复。

**2. ⚡ 秒杀思路**
* **配置管理方案：**
    * **手动检查:** 效率低，无法持续监控。
    * **脚本检查:** 维护困难，覆盖面有限。
    * **Config Rules:** 自动化配置合规检查。
* **Config Rules 特性：**
    * **持续监控:** 实时检查资源配置。
    * **托管规则:** 丰富的预定义规则库。
    * **自定义规则:** 支持自定义检查逻辑。
    * **自动修复:** 自动纠正配置问题。
* **合规优势：**
    * **选项 A (CloudTrail):** 仅记录 API 调用，不检查配置。
    * **选项 B (定期审计):** 反应式，无法及时发现问题。
    * **选项 C (Config Rules):** 
        * 持续配置监控。
        * 自动合规评估。
        * 支持自动修复。
    * **选项 D (无检查):** 配置漂移，合规风险。
* **配置策略：**
    * 选择合适的托管规则。
    * 开发自定义规则满足特殊需求。
    * 配置自动修复和通知。
    * 建立合规报告和仪表板。

**3. ✅ 正确选项解析 (选项 C)**
* **AWS Config Rules:** 自动化配置合规检查服务，确保资源符合企业标准。

**5. 📚 核心考点:** Config Rules 在基础设施治理中的应用和最佳实践。

#### 📝 [282/529] RDS Custom Oracle 企业级特性支持

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** Oracle E-Business Suite 部署。
* **需求：** 访问底层操作系统，安装特定补丁。
* **约束：** 需要管理员权限，自定义 Oracle 配置。
* **当前：** 标准无法满足权限需求。

**2. ⚡ 秒杀思路**
* **RDS 版本选择：**
    * **RDS Standard:** 完全托管，无 OS 访问权限。
    * **RDS Custom:** 
        * 提供操作系统访问权限。
        * 支持自定义 Oracle 配置。
        * 允许安装特定补丁。
        * 兼容企业级应用。
* **关键特性：**
    * **选项 A (RDS Standard):** 权限不足。
    * **选项 B (EC2 + Oracle):** 自管理，运维复杂。
    * **选项 C (RDS Custom):** 
        * 托管服务 + 自定义权限。
        * 支持 E-Business Suite。
        * 降低运维负担。
    * **选项 D (RDS on Outposts):** 本地部署，不解决权限问题。
* **选择：** RDS Custom 是专门为这类场景设计的。

**3. ✅ 正确选项解析 (选项 C)**
* **RDS Custom:** 提供企业级 Oracle 部署所需的灵活性和控制权。

**5. 📚 核心考点:** RDS Custom 的使用场景与传统 RDS 的区别。

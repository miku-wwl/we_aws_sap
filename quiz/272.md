#### 📝 [272/529] API 网关直接与 SQS 集成

**1. 🕵️‍♂️ 题眼与约束分析**
* **场景：** 移动应用通过 API Gateway 上传数据。
* **后端：** SQS 队列，EC2 消费者处理。
* **需求：** 简化架构，降低管理开销。
* **当前：** API Gateway -> Lambda -> SQS -> EC2。

**2. ⚡ 秒杀思路**
* **AWS 集成模式：** API Gateway 支持 **直接服务集成**，无需 Lambda 中转。
* **集成选项：**
    * **选项 A (Lambda):** 现有架构，有 Lambda 运行成本和管理开销。
    * **选项 B (直接集成 SQS):** 
        * API Gateway 可以直接配置为 SQS 的 **HTTP 代理**。
        * 使用 AWS Integration 类型，配置 SQS 的 `SendMessage` API。
        * 移除 Lambda，简化架构。
    * **选项 C (Kinesis):** 不符合现有 SQS 架构。
    * **选项 D (Step Functions):** 增加复杂度。
* **选择：** B 是最佳简化方案。

**3. ✅ 正确选项解析 (选项 B)**
* **API Gateway Direct Integration:** 直接调用 SQS API，消除 Lambda 中转层。

**5. 📚 核心考点:** API Gateway 服务集成模式，架构简化最佳实践。

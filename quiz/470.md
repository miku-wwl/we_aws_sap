#### 📝 [470/529] 集中式防火墙部署 (GWLB)

**1. 🕵️‍♂️ 题眼与约束分析**
* **架构:** 多账户 (Centralized networking, Member accounts)。
* **需求:** **Third-party network firewall appliances** (第三方防火墙)。
* **目标:** Centralized solution for inspection (集中检查)。
* **选项:** 选择 3 个。

**2. ⚡ 秒杀思路**
* **防火墙负载均衡：**
    * **Gateway Load Balancer (GWLB):** 专为防火墙、入侵检测等网络 appliances 设计，透明流量转发。
    * $→$ **选中 A**。
* **防火墙扩展：**
    * **Auto Scaling Group (C):** 防火墙设备放在 ASG 中，实现水平扩展。
    * $→$ **选中 C**。
* **流量入口：**
    * **GWLB Endpoint (F):** 在集中网络账户创建 GWLB Endpoint，成员账户通过 TGW 或 VPC 路由将流量发送到 Endpoint。
    * $→$ **选中 F**。
* **选项对比：**
    * A (GWLB): 正确，专为 appliances 设计。
    * B (NLB): GWLB 内部使用 NLB，但对外应该用 GWLB。
    * C (ASG): 正确，防火墙需要扩展。
    * D/E (Route 53 Resolver): DNS 转发，与流量检查无关。
    * F (GWLB Endpoint in Centralized account): 正确，集中管理 Endpoint。
* **锁定 A, C, F。**

**3. ✅ 正确选项解析 (选项 A, C, F)**
* **GWLB:** 专为防火墙设备设计的负载均衡器。
* **ASG:** 实现水平扩展。
* **GWLB Endpoint:** 流量入口。

**5. 📚 核心考点:** Gateway Load Balancer 在集中式网络安全架构中的应用。

#### 📝 [234/529] 本地 Windows 灾备到 AWS (DRS + DataSync)

**1. 🕵️‍♂️ 题眼与约束分析**
* **源端：** Hundreds of Windows Servers + **Common Share** (公共共享存储)。
* **RTO/RPO:** 15 min / 5 min。
* **需求：** On-premise Failover and **Failback** (故障切换与回切)。
* **成本：** **Cost-effectively**。

**2. ⚡ 秒杀思路**
* **服务器灾备：**
    * **AWS Elastic Disaster Recovery (DRS):** (D 选项)。支持物理机/VM 到 AWS 的块级复制，RPO 秒级，RTO 分钟级。支持 **Failback**（增量回切）。完美。
    * 选项 A (Storage Gateway): 只是备份数据，不是服务器灾备。恢复服务器需要重装系统，RTO 达不到 15 分钟。
    * 选项 B (CloudFormation + DataSync): DataSync 是文件级的，不能复制操作系统状态。你需要预先维护几百个 CloudFormation 模板来重建服务器？太慢太复杂。
    * 选项 C (CDK + S3 Sync): 同上，S3 Sync 不能同步操作系统。
* **共享存储灾备：**
    * 本地的 "Common Share" 需要在 AWS 有对应物。
    * **Amazon FSx for Windows File Server** 是最佳对应。
    * 使用 **DataSync** 将本地共享复制到 FSx。
* **组合：**
    * Server -> DRS。
    * Share -> DataSync -> FSx。
    * **锁定 D。**

**3. ✅ 正确选项解析 (选项 D)**
* **AWS DRS:** 解决服务器（OS+App）的低 RPO/RTO 灾备与回切。
* **DataSync + FSx:** 解决共享文件数据的同步。

**5. 📚 核心考点:** 完整的企业级灾备方案 (DRS + DataSync)。
